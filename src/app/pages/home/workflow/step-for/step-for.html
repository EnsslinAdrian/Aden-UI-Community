Step 04

Design Tokens & Styling

Maintain UI consistency. Never use hardcoded colors. Always define your palette in the :host selector.

component.scss
:host {
--aden-primary-green: #4ade80;
--aden-surface-dark: #1a1a1a;
--aden-custom-glow: rgba(74, 222, 128, 0.2);
}
.my-box {
background: var(--aden-surface-dark);
}

The Source: _variables.scss

Before styling, check the global variable file. Copy the tokens you need into your component's :host.

The --aden- Prefix

If you need a color that isn't in the global file, you must prefix it with --aden-. This ensures compatibility with the entire library.

No Hardcoded Values

Avoid using raw hex or rgb values inside your classes. Reference them via var() to keep the component themeable.
